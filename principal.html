Guía Completa de Desarrollo Móvil

Guia de uso de desarrolo movil

Sistemas Operativos Móviles

Android: arquitectura Linux con runtime ART. iOS: kernel XNU, ecosistema cerrado. Otros: HarmonyOS, KaiOS.

Componentes centrales

  • Kernel y drivers
  • Gestión de memoria
  • Gestión de procesos y hilos
  • Gestión de permisos
  • Servicios de sistema (notificaciones, sensores, ubicaciones)

Tipos de Apps

Nativas

Compiladas para la plataforma. Ventajas: rendimiento y acceso a APIs nativas. Lenguajes: Kotlin/Java, Swift/Objective-C.

Híbridas / Multiplataforma

Usan un contenedor nativo para ejecutar web code o código cross-platform. Ejemplos: Ionic, Cordova, Capacitor, React Native, Flutter.

Progressive Web Apps (PWA)

Aplicaciones web que se comportan como apps instalables mediante manifest y service workers.

Hardware Móvil

  • SoC: CPU (ARM), GPU
  • RAM y almacenamiento flash
  • Batería y gestión energética
  • Sensores: acelerómetro, giroscopio, magnetómetro, GPS, proximidad, luz ambiental, barómetro, fingerprint
  • Conectividad: Wi-Fi, Bluetooth, NFC, LTE/5G

Lenguajes y Entornos

Kotlin/Java (Android), Swift (iOS), Dart (Flutter), JavaScript/TypeScript (Ionic, React Native). Entornos: Android Studio, Xcode, VS Code.

Diseño UI y Guías de Estilo

Material Design y Human Interface Guidelines. Principios: jerarquía visual, accesibilidad, consistencia, feedback claro, tamaño de hit target y adaptabilidad a diferentes pantallas.

Arquitecturas y Patrones

MVC, MVP, MVVM, Clean Architecture. Separar capas: presentación, dominio y datos. Pruebas unitarias, integraciones y pruebas de UI.

Control de Versiones

Git: commits atómicos, ramas (feature, develop, release), Pull Requests, revisión de código y tags para releases. Repositorios: GitHub, GitLab, Bitbucket.

Persistencia y Acceso a Datos

Local

localStorage, IndexedDB, SQLite (nativo), Room (Android), CoreData (iOS).

Remoto

APIs REST, GraphQL, Firebase Realtime/Firestore. Estrategias: caching, sincronización, conflictos de sincronización.

Gestión de Sensores

Los sensores se acceden mediante APIs expuestas por el navegador o SDK nativo. A continuación se explican los más relevantes y ejemplos de uso.

Geolocalización (GPS)

Permite obtener latitud/longitud y seguimiento. Ejemplo de uso en web: navigator.geolocation.getCurrentPosition y watchPosition.

Acelerómetro y Giroscopio

Detectan movimiento y orientación. En web existen eventos DeviceMotionEvent y DeviceOrientationEvent; también hay la Generic Sensor API cuando está disponible.

Sensor de Luz y Proximidad

Se usan para ajustar brillo o detectar proximidad durante llamadas.

Gestión práctica

  • Solicitar permisos en tiempo de ejecución.
  • Normalizar valores y filtrar ruido.
  • Optimizar muestreo para ahorrar energía.

Servicios y Procesos en Segundo Plano

Servicios permiten ejecutar tareas sin interfaz: sincronización, reproducir audio, descargar archivos, location tracking. En web se usan service workers, Background Sync y Web Push para ciertos casos.

Buenas prácticas

  • Minimizar uso de CPU en background.
  • Usar trabajo por lotes.
  • Manejar fallos de red y reintentos exponenciales.

Demo: registrar permiso de notificaciones

Notificaciones

Locales (generadas por la app) y push (enviadas desde servidor). Elementos: título, cuerpo, icono, acciones y datos. En web se usan Notification API y Web Push con un servidor que envía mensajes vía VAPID.

Flujo Push (resumen)

  1. Usuario concede permiso y se subscribe al Push Service.
  2. Servidor almacena la suscripción y envía mensajes vía Push Service (p. ej. Firebase Cloud Messaging o un servidor propio con VAPID).
  3. Service Worker recibe el push y muestra la notificación.

Buenas prácticas

  • No abusar de notificaciones
  • Permitir control granular en la app
  • Proveer deep-links en notificaciones

Ciclo de Vida de las Apps

Estados principales: creado, activo, en pausa/background, detenido y destruido. En Android: onCreate, onStart, onResume, onPause, onStop, onDestroy. En iOS: applicationDidEnterBackground, applicationWillEnterForeground, etc.

Consideraciones

  • Preservar estado
  • Guardar datos en onPause/onStop
  • Reanudar conexiones y sensores al volver a primer plano

Conectividad

Detectar cambios de red y adaptarse: navigator.onLine y eventos online/offline. Estrategias: modo offline-first, caché, reintentos y sincronización en background.

APIs Nativas Relevantes

  • Geolocation API
  • DeviceMotion / DeviceOrientation
  • Generic Sensor API
  • Web Bluetooth / Web USB (soporte limitado)
  • Payment Request API
  • Camera & MediaDevices.getUserMedia

Seguridad y Privacidad

Principios: mínimo privilegio, encriptación en tránsito y reposo, no almacenar secretos en el cliente, revisar bibliotecas de terceros, usar HTTPS y almacenamiento seguro.

Empaquetado y Despliegue

Generar APK/IPA o publicar PWA. Certificados, firmas y cumplimiento de políticas de stores (privacidad, permisos, contenido).

Referencias y Recursos

  • Google Developers (Material Design, Android)
  • Apple Developer (Human Interface Guidelines)
  • Documentación de Ionic, React Native y Flutter

Ejercicios sugeridos

  1. Implementa un sistema de sincronización que guarde datos localmente y sincronice al reconectarse.
  2. Crear notificaciones push con FCM y un pequeño servidor de ejemplo.
  3. Registrar y mostrar datos de sensores para una función específica (p. ej. conteo de pasos).
